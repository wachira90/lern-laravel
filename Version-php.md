สรุปเวอร์ชัน **PHP** ที่รองรับ (Compatible) กับ **Laravel** เวอร์ชันต่างๆ

### ตารางความสัมพันธ์ระหว่าง Laravel และ PHP

| Laravel Version | PHP Version (ขั้นต่ำ - สูงสุดที่แนะนำ) | สถานะปัจจุบัน |
| --- | --- | --- |
| **Laravel 11.x** | **8.2** – 8.3 | ล่าสุด (Stable) |
| **Laravel 10.x** | 8.1 – 8.3 | ยังสนับสนุนด้าน Security |
| **Laravel 9.x** | 8.0 – 8.2 | สิ้นสุดการสนับสนุน (EOL) |
| **Laravel 8.x** | 7.3 – 8.1 | สิ้นสุดการสนับสนุน (EOL) |
| **Laravel 7.x** | 7.2.5 – 8.0 | สิ้นสุดการสนับสนุน (EOL) |
| **Laravel 6.x (LTS)** | 7.2 – 8.0 | สิ้นสุดการสนับสนุน (EOL) |

### ข้อแนะนำเพิ่มเติมในการเลือกเวอร์ชัน

* **เน้นความปลอดภัย:** หากคุณกำลังเริ่มโปรเจกต์ใหม่ แนะนำให้ใช้ **Laravel 11** ร่วมกับ **PHP 8.2 หรือ 8.3** เนื่องจากเป็นเวอร์ชันล่าสุดที่มีการอัปเดตความปลอดภัยสม่ำเสมอ
* **การอัปเกรด:** หากปัจจุบันคุณใช้ PHP เวอร์ชันที่เก่ากว่า 8.1 (เช่น 7.4) และต้องการอัปเกรด Laravel คุณจำเป็นต้องอัปเกรดเวอร์ชันของ PHP ใน Server หรือ Container ของคุณก่อน
* **Extension:** อย่าลืมตรวจสอบว่า PHP extension ที่ Laravel ต้องการ (เช่น `bcmath`, `ctype`, `fileinfo`, `openssl`) ถูกติดตั้งครบถ้วนตามความต้องการของเวอร์ชันนั้นๆ

> **Note:** สำหรับการใช้งานในระดับ Production แนะนำให้ใช้ PHP เวอร์ชันที่ยังอยู่ในช่วง **Active Support** หรือ **Security Support** เท่านั้น เพื่อป้องกันช่องโหว่ทางไซเบอร์ครับ
